ブランチ作成練習

🌿 練習1: 基本的なブランチ操作​

ステップ1: 新しいブランチを作成​

  1. GitHubでリポジトリを開く
  2. 「main」と書かれたドロップダウンをクリック
  3. practice/your-nameと入力（your-nameは自分の名前に置き換える）
  4. 「Create branch: practice/your-name」をクリック

ステップ2: ブランチで作業​

  1. 新しいファイル introduction.md を作成
  2. 以下の内容を記入:
# 自己紹介

## 名前
[あなたの名前]

## 好きなこと
- 
- 
- 

## GitHubを学ぶ理由
[なぜGitHubを学びたいか書いてください]
  1. 「Commit new file」でコミット

ステップ3: ブランチを切り替えて確認​

  1. ブランチドロップダウンから「main」を選択
  2. ファイル一覧を確認（introduction.mdがないことを確認）
  3. 再度自分のブランチに戻る
  4. introduction.mdがあることを確認

🔄 練習2: プルリクエストの作成​

ステップ1: 変更を加える​

  1. 自分のブランチで practice-log.md を作成
  2. 以下の内容を記入:
# GitHub練習ログ

## 今日学んだこと
- ブランチの作成方法
- ブランチの切り替え方法
- ファイルの追加方法

## 感想
[学んでみての感想を書いてください]

ステップ2: プルリクエストを作成​

  1. リポジトリのトップページに戻る
  2. 「Compare & pull request」ボタンをクリック
  3. タイトル: 「練習: [あなたの名前]の自己紹介とログを追加」
  4. 本文に以下を記入:
## 追加した内容
- 自己紹介ファイル (introduction.md)
- 練習ログファイル (practice-log.md)

## 確認事項
- [x] ファイルが正しく追加されている
- [x] マークダウンの書式が正しい
- [x] 個人情報は含まれていない
  1. 「Create pull request」をクリック

⚔️ 練習3: コンフリクトの解決​

準備: コンフリクトを作る​

  1. mainブランチで favorite-foods.md を作成:
# 好きな食べ物リスト

## 和食
- 寿司
- 天ぷら
- うどん
  1. branch-a を作成して同じファイルを編集:
# 好きな食べ物リスト

## 和食
- 寿司
- 天ぷら
- そば  ← うどんをそばに変更
  1. branch-b を作成して同じファイルを編集:
# 好きな食べ物リスト

## 和食
- 寿司
- 天ぷら
- ラーメン  ← うどんをラーメンに変更

コンフリクトの解決​

  1. branch-a からPRを作成してマージ
  2. branch-b からPRを作成（コンフリクトが発生）
  3. 「Resolve conflicts」ボタンをクリック
  4. 以下のように編集:
# 好きな食べ物リスト

## 和食
- 寿司
- 天ぷら
- そば
- ラーメン  ← 両方残す
  1. 「Mark as resolved」→「Commit merge」

📝 チェックリスト​

ブランチ操作​

  • 新しいブランチを作成できた
  • ブランチを切り替えられた
  • ブランチごとに異なるファイルがあることを確認できた

プルリクエスト​

  • PRを作成できた
  • PR の説明を書けた
  • マージできた

コンフリクト​

  • コンフリクトを意図的に作れた
  • Web上でコンフリクトを解決できた
  • 解決後にマージできた

💡 ヒント​

ブランチ名の付け方​

良い例:
- feature/user-profile     (機能追加)
- fix/login-bug           (バグ修正)
- docs/api-reference      (ドキュメント)
- chore/update-deps       (メンテナンス)

避けるべき例:
- test123
- my-work
- aaa

コミットメッセージの書き方​

良い例:
- 「ユーザープロフィール機能を追加」
- 「ログインエラーを修正」
- 「READMEに使用方法を追記」

避けるべき例:
- 「更新」
- 「修正」
- 「ファイル追加」

PRの説明の書き方​

  1. 何を変更したか明確に書く
  2. なぜ変更が必要だったか説明する
  3. どのように確認したか記載する
  4. 関連する情報があればリンクする
